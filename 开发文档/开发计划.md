## 三、UI 设计

### 3.1 整体布局
```
┌─────────────────────────────────────────────────────────────────┐
│  [手动更新] [备份] [传输预览] [传输] [收藏夹 ▼]                    │  ← 顶部按钮区
├───────────────────┬───────────────────┬─────────────────────────┤
│    本地文件列表    │   传输差异列表     │     SVN 文件列表        │  ← 列表对比区
│                   │                   │                         │
│   (支持拖拽)       │  🔴 有修改        │    (支持拖拽)           │
│                   │  🟢 无修改        │                         │
│                   │  🟠 SVN较新       │                         │
│                   │                   │                         │
├───────────────────┴───────────────────┴─────────────────────────┤
│   本地预览图         │        SVN 预览图                            │  ← 预览图区域
│   (棋盘格背景)        │      (棋盘格背景)                            │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 顶部按钮区
| 按钮 | 功能 |
|------|------|
| 手动更新 | 刷新左右两侧文件列表 |
| 备份 | 将当前 SVN 目录备份到指定位置（压缩包） |
| 传输预览 | 显示即将传输的文件清单，二次确认 |
| 传输 | 执行文件复制 + SVN 提交 |
| 收藏夹 ▼ | 下拉菜单，显示历史/预设目录 |

### 3.3 列表对比区
| 区域 | 功能 | 显示字段 |
|------|------|---------|
| 左侧（本地） | 显示本地文件，支持拖拽文件夹 | 文件名、路径、修改时间 |
| 中间（差异） | 显示对比结果 | 文件名、状态（🔴修改/🟢相同/🟠SVN较新） |
| 右侧（SVN） | 显示 SVN 文件，支持拖拽 | 文件名、路径、修改时间 |

### 3.4 预览图区域
- **位置**：左右列表下方各自显示
- **格式支持**：png、jpg、gif
- **大文件限制**：超过 10MB 不加载预览
- **显示方式**：
  - 默认：透明区域保持透明
  - 提供按钮切换棋盘格背景显示
  - 自动缩放最大化但不变形

## 四、功能模块

### 4.1 文件扫描模块
- 扫描指定文件夹下的所有文件
- 获取文件属性：修改时间、文件大小
- 图片文件额外获取：像素尺寸
- 支持过滤规则（忽略 .svn、.git、__pycache__ 等，可扩展）

### 4.2 文件对比模块
#### 对比规则
| 条件 | 结果 |
|------|------|
| 文件大小不同 | 🔴 本地已修改 |
| 修改时间不同 | 🔴 本地已修改 |
| 文件仅存在于本地 | 🔴 新文件 |
| 文件仅存在于 SVN | 🟠 SVN 较新 |
| 两边一致 | 🟢 无需更新 |

#### 图片文件额外对比
- 像素尺寸不同 → 🔴 本地已修改

### 4.3 历史记录模块
- 自动记忆访问过的文件夹对（最多 30 个）
- 持久化保存（JSON 文件）
- 软件启动时自动加载上次使用的文件夹对
- 软件切换到前台时自动刷新本地列表、检测 SVN 更新

### 4.4 收藏夹模块
- 预设目录：
  1. `E:\XYJProject\美术资源\动画序列帧\角色输出图`
  2. `D:\Work\预传输前序列帧`
- 支持添加/删除/编辑收藏项
- 收藏夹入口：顶部按钮区最后一个

### 4.5 备份模块
- 备份到指定目录
- 压缩包格式（zip）
- 自动命名规则：`文件夹名_时间戳.zip`
- 示例：`角色输出图_20260528_143025.zip`

### 4.6 SVN 操作模块
#### SVN 更新
- 调用命令：`TortoiseProc.exe /command:update /path:"路径"`
- 或命令行：`svn update`

#### SVN 提交流程
1. 点击「传输」按钮
2. 弹出对话框显示变更清单
3. 自动生成提交信息（可编辑）：
   ```
   【标题】更新 N 个文件
   【详情】
   - 新增：file1.png, file2.png
   - 修改：file3.png
   - 删除：file4.png
   ```
4. 用户确认/修改后提交
5. 执行：复制文件 → `svn add` → `svn commit`

### 4.7 冲突处理
- 检测到本地和 SVN 都有修改时
- 弹出警告提示
- 显示推荐解决方案（保留本地/保留SVN/手动合并）

### 4.8 文件过滤模块
- 默认过滤规则：
  ```
  .svn/
  .git/
  __pycache__/
  *.pyc
  .DS_Store
  Thumbs.db
  ```
- 支持用户自定义扩展过滤规则
